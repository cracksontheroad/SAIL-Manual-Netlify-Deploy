What is SuperTeacher?

SuperTeacher is a standalone AI-powered classroom management and instruction tool designed for individual teachers. It requires no school infrastructure, integration setup, or IT involvement—just launch and teach.

Key Characteristics

Standalone Architecture

SuperTeacher is self-contained. You don't need a school learning management system, API keys, or approval from an IT department. Everything—students, courses, assignments, grades, content, and analytics—lives within SuperTeacher.

AI-Powered Across the Board

From lesson planning to grading to parent communication, AI capabilities are embedded throughout SuperTeacher. You control when to use them.

Teacher-First Design

Every feature is designed for teachers. Student roster, grade entry, attendance tracking, wellbeing monitoring, and assignment creation are all streamlined for speed and clarity.

Comprehensive Student Insights

Monitor academic performance (grades, trends, completion), behavior (engagement, attendance), and wellbeing (mood check-ins, flagged entries) all in one dashboard.

What's Not Included

SuperTeacher is not:

  • A student-facing learning platform (students access content via links/downloads)
  • Integrated with school SIS/MIS systems
  • A parent portal (teachers generate and share reports manually)
  • A communication hub (basic internal messaging only)
  • A storage service (users manage their own file backups)

Ideal For

SuperTeacher is perfect for teachers who want a complete, standalone classroom management solution without vendor lock-in or school infrastructure dependencies. Works for K-12 through university.

Join Code System

The join code system lets students self-register into your classroom without you manually adding each one. It's ideal for large classes or remote students.

How Join Codes Work

  1. You generate a unique code (e.g., ABC123) that's valid for 30 days
  2. You share it with students (verbally, email, classroom wall, QR code)
  3. Students use the code to register and join your classroom
  4. Once registered, they appear in your roster automatically

Generating a Join Code

1
Go to Student Roster
My Classroom → Student Roster
2
Click "Generate Join Code"
A modal appears showing your code and expiration date.
3
Copy or Download
Click "Copy" to copy the code to clipboard. Click "Download QR Code" to get a printable QR code image.
4
Share with Students
Email the code, display on a projector, print the QR code, or add to your syllabus.

Viewing Active Join Codes

On the Student Roster page, you'll see a section listing all active join codes (not yet expired). Each shows:

  • The code itself
  • Created date
  • Expiration date (always 30 days out)
  • Number of students who've used it
  • Status: Active or Expired

Regenerating a Join Code

If you want to invalidate an old code and create a new one (e.g., end of term, class split):

1
Find the Code
In the Join Codes section, locate the code you want to replace.
2
Click "Regenerate"
The old code becomes invalid immediately. A new code is generated and shown.
3
Share the New Code
Update your classroom wall, email, etc. with the new code.
Regenerating does not remove students who already used the old code. They remain in your roster. Regeneration only prevents future students from using the old code.

Join Code Security

Join codes are classroom-specific and single-use per student. Once a student uses a code to register, they have an account. The code itself can be shared publicly—it only creates an account, not access to data.

Troubleshooting Join Codes

Student says the code is invalid ▼

Check: Has the code expired? (30-day limit). Did you accidentally regenerate it? Is the student entering it exactly as shown (case-sensitive)?

Solution: Generate a new code and share it with the student again.

Student registered but appears as "Enrolled" instead of "Active" ▼

This is normal. Status changes from "Enrolled" to "Active" once the student logs in for the first time and accesses their dashboard.

I want to add a student directly without a join code ▼

Solution: Use the Manual Entry method. Go to My Classroom → Student Roster → Add Student and fill in their details directly.

What Does Selecting a Framework Do?

AI Pre-Grading

SuperTeacher can automatically score submissions against your rubric using AI. You review and override any scores as needed.

How AI Pre-Grading Works

  1. You create an assignment with a rubric
  2. Toggle "Enable AI Pre-Grading" when creating the assignment
  3. Students submit work (essays, reports, answers, etc.)
  4. SuperTeacher automatically scores each submission against each rubric criterion
  5. You review the AI scores and grades (takes seconds per submission)
  6. You can accept, override, or ask AI to explain its reasoning
  7. Final grade is recorded in the gradebook

Enabling AI Pre-Grading

1
Create Assignment with Rubric
Follow the assignment creation steps and attach a rubric.
2
Toggle "Enable AI Pre-Grading"
In the grading section, check the box "Use AI to automatically score submissions".
3
Configure AI Settings
Rubric Model: How strictly should AI interpret criteria? (Lenient, Balanced, Strict)
Auto-Publish Scores? If yes, grades are immediately visible to students. If no, you review first.
4
Create Assignment
Students submit; AI begins scoring automatically.

Understanding the Three Versions

Foundation Version

For students working below grade level or needing support. Features:

  • Simplified prompts with scaffolding (step-by-step questions)
  • Additional resources or examples provided
  • Shorter length requirement (e.g., 3 paragraphs instead of 5)
  • Vocabulary support (definitions provided)

Standard Version

For students at grade level. The standard expectation.

  • Grade-level prompt with clear expectations
  • Typical length and complexity
  • Resources available but not spoon-fed

Extension Version

For advanced students. Features:

  • Complex prompt with higher-order thinking (analyze, synthesize, evaluate)
  • Longer response (e.g., 8-10 pages for a report)
  • Independent research expected
  • Application or creative twist (connect to real-world, create original product)

Grading Differentiated Assignments

In the gradebook, all three versions appear as a single assignment. You grade each student on their version using the same rubric (or different rubrics if you prefer).

Differentiation helps all students succeed: Foundation students aren't overwhelmed, Standard students meet expectations, Extension students are challenged.

Example: Biology Lab Report

Topic: "Investigate the effect of light on plant growth"

  • Foundation: Observation worksheet with guiding questions, picture labels of plant parts, conclusion is 1-2 sentences.
  • Standard: Full lab report (intro, methods, results, discussion), 5-7 pages, analyze data with graphs.
  • Extension: Full report + design a follow-up experiment testing a different variable, predict outcomes, cite 3+ scientific sources.
All three versions should teach the same concept but at different depths. Don't assign "different" content; scaffold the same content at different complexity levels.
